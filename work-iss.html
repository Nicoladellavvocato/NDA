A real-time tracker for the International Space Station, built directly into this portfolio. It reads live orbital data, computes where the station is over the Earth, and plots its position, trail and visibility footprint on a world map — refreshing continuously.

01 Concept

The tracker grew out of my work computing satellite positions in Python with skyfield, propagating orbits from TLE data using the SGP4 model. Having worked out the orbital mechanics offline, I wanted a version that was live, visual and embedded somewhere people could actually see it — so I rebuilt the idea as a lightweight in-browser component for this site.

02 How it works

On a fixed interval the tracker pulls the station's current orbital state and derives the quantities shown in the telemetry panel:

  • Sub-point — the latitude and longitude directly beneath the ISS, i.e. where it is over the ground.
  • Altitude & ground speed — how high it is and how fast its sub-point sweeps across the surface.
  • Sunlight — whether the station is currently in daylight or in Earth's shadow.

Those coordinates are drawn onto an equirectangular projection — a simple, direct mapping of longitude to the horizontal axis and latitude to the vertical, which is why the world map is a plain rectangle with a graticule.

03 The trail & footprint

As updates arrive, recent sub-points are joined into a ground trail behind the marker. Because the map wraps at ±180° longitude, the trail handles the antimeridian crossing so the line doesn't streak straight across the map when the station passes the date line. A translucent footprint circle around the marker indicates the region of the Earth that has the station above its horizon.

04 Telemetry readout

Alongside the map, a live telemetry block lists latitude, longitude, altitude, ground speed, sunlight state and the current time, updated in step with the position so the numbers and the marker always agree.

05 Resilience

Because the data comes from a live external source, the component is built to fail gracefully. A status indicator reflects the link state, and if a fetch fails the tracker shows signal lost and keeps retrying on its refresh cycle rather than freezing or breaking the page. The whole feature is also written to degrade quietly: if the component isn't present or data never arrives, the rest of the site is unaffected.

06 From Python to the browser

The interesting part of the project was translating orbital-mechanics reasoning I'd done in Python into something live and visual on the web — the same underlying ideas of orbital state, sub-point and propagation, repackaged as a small, robust, self-contained piece of front-end engineering. It's the signature interactive element of this portfolio and a compact demonstration of taking an idea end-to-end.
